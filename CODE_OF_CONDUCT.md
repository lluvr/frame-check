# Code of Conduct

Frame Check is a public research program. Participation, whether
through code contributions, frame-library candidates, validation
runs, methodology critique, or issue conversations, is open to
anyone willing to engage in good faith with the substance of the
work.

## What we expect

- **Engage with the substance.** Critique of measurement, methodology,
  or contract design is the heart of the program. Surface the
  reasoning, not the conclusion. "This rule misfires on these inputs"
  beats "this rule is wrong"; the former is testable.
- **Cite what you read.** When you reference a worked example,
  validation run, frame card, or external source, link it. Honest
  attribution is the same discipline the tool itself enforces.
- **Honest limits.** State what you tested, what you did not test,
  and what would change your mind. "I checked X but not Y" is more
  valuable than an unbounded claim.
- **Reviewer time is finite.** Read `CONTRIBUTING.md` before opening
  a substantive PR; read `GOVERNANCE.md` before re-litigating a
  governance decision. Both exist to make the program legible.
- **Disagree under your own name.** A signed-off-by-DCO commit is
  the contributing convention; the same standing applies to
  substantive issue and PR comments. If a contribution is meant to
  carry weight in a methodology decision, the author needs to be
  reachable.

## What is not welcome

- Personal attacks, harassment, threats, or insults directed at
  contributors, reviewers, raters, or any third party.
- Content that targets people based on protected characteristics
  (race, sex, gender identity or expression, sexual orientation,
  national origin, religion, disability, age).
- Sustained derailing of substantive review threads with
  off-topic content.
- Doxing, sharing private contact information, or attempts to
  pressure individual contributors outside the public review
  surface.
- Claims about Frame Check's measurements, calibration, or
  methodology that the claimant has not actually run against the
  artifact at the version they reference.

## Scope

This Code of Conduct applies to:

- The `Clarethium/frame-check` GitHub repository (issues, PRs,
  discussions, code review threads, commit messages).
- The associated PyPI project page (`frame-check-mcp`).
- Any conversation about Frame Check on the maintainer's blog
  (`blog.clarethium.com`) or public talks where the maintainer
  is hosting or moderating.
- Coordinated cross-repo work that spans Frame Check and the
  methodology canon at `Clarethium/lodestone`.

It does not extend to private channels, third-party platforms, or
the maintainer's other unrelated projects.

## Reporting

Report Code of Conduct violations to the maintainer at
`lovro.lucic@gmail.com`. Reports are read by the maintainer alone;
the maintainer does not delegate triage to bots or automated
systems. Expect a first response within seven days.

In the report:

- Describe what happened and where (issue / PR / commit URL).
- Note any preceding interactions if context matters.
- State what outcome you are looking for (mediation, public
  response, removal of content, contributor block, etc.).

The maintainer will read every report carefully. Frivolous reports
or reports filed in bad faith are themselves Code of Conduct
violations.

## Enforcement

The maintainer has authority to:

- Edit, mark as off-topic, or hide issue or PR comments that
  violate this Code of Conduct.
- Lock issue or PR threads where conversation has degraded to a
  point where substantive review is no longer possible.
- Block contributors whose participation, after notice, continues
  to violate this Code of Conduct.

Enforcement actions on individual incidents are not subject to
public review thread; the maintainer's decision is the appeal of
last resort. The reasoning behind a non-trivial enforcement
decision will be made available to the affected contributor on
request.
